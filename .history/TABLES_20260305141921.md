# Tables de la Base de Données - KilysAgri

## Liste des Tables

### 1. **user** (Utilisateur)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| email | String (180) | Email unique |
| password | String | Mot de passe hashé |
| nom_complet | String (255) | Nom complet |
| telephone | String (20) | Numéro de téléphone |
| roles | JSON | Rôles (ROLE_USER, ROLE_VENDEUR, ROLE_ADMIN) |
| is_verified | Boolean | Compte vérifié (email) |
| is_vendeur | Boolean | Peut vendre des produits |
| is_kyc_validated | Boolean | KYC validé par admin |
| confirmation_code | String (6) | Code de vérification |
| code_expires_at | DateTime | Expiration du code |

**Relations:**
- 1 user → 0..* Kyc (soumet)
- 1 user → 0..* Produit (publie)
- 1 user → 0..* Commande (passe)
- 1 user → 0..* Exploitation (possede)
- 1 user → 0..* Notification (recoit)
- 1 user → 0..* Conseil (redige)
- 0..1 user → 1 Administrateur (est_un)

---

### 2. **kyc** (Vérification d'identité)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| id_utilisateur | Integer | FK vers user |
| type_piece | String (50) | Type: cni, passeport, permis |
| numero_piece | String (100) | Numéro de la pièce |
| photo_piece_recto | String (255) | Photo recto |
| photo_piece_verso | String (255) | Photo verso |
| photo_selfie | String (255) | Photo selfie avec pièce |
| statut | String (20) | en_attente, valide, rejete |
| created_at | DateTime | Date de soumission |

**Cardinalité:** 1 utilisateur → 0..* KYC

---

### 3. **categorie** (Catégorie de produits)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| nom | String (255) | Nom de la catégorie |
| description | String | Description |

**Cardinalité:** 1 catégorie → 0..* produit

---

### 4. **produit** (Produit agricole)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| id_utilisateur | Integer | FK vers user (vendeur) |
| categorie_id | Integer | FK vers categorie |
| nom | String (255) | Nom du produit |
| description | Text | Description |
| prix | Decimal (10,2) | Prix unitaire |
| quantite | Integer | Quantité en stock |
| image | String (255) | Chemin de l'image |
| created_at | DateTime | Date de création |
| updated_at | DateTime | Date de modification |

**Cardinalités:**
- 1 utilisateur → 0..* produit
- 1 catégorie → 0..* produit

---

### 5. **commande** (Commande client)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| id_utilisateur | Integer | FK vers user (client) |
| montant_total | Decimal (10,2) | Montant total |
| statut | String (20) | en_attente, payee, en_cours, livree, annulee |
| date_commande | DateTime | Date de la commande |

**Cardinalités:**
- 1 utilisateur → 0..* commande
- 1 commande → 0..* ligne_commande
- 1 commande → 0..1 paiement

---

### 6. **ligne_commande** (Ligne de commande)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| commande_id | Integer | FK vers commande |
| produit_id | Integer | FK vers produit |
| quantite | Integer | Quantité commandée |
| prix_unitaire | Decimal (10,2) | Prix au moment de la commande |

**Cardinalités:**
- 1 commande → 0..* ligne_commande
- 1 produit → 0..* ligne_commande

---

### 7. **paiement** (Paiement)
| Colonne | Type | Description |
|---------|------|-------------|
| id | Integer | Clé primaire |
| commande_id | Integer | FK vers commande |
| montant | Decimal (10,2) | Montant payé |
| reference | String | Référence du paiement |

| date_paiement | DateTime | Date du paiement |

## Résumé des Cardinalités

| Relation | Cardinalité |
|----------|--------------|
| User → Kyc | 1 : 0..* |
| User → Produit | 1 : 0..* |
| User → Commande | 1 : 0..* |
| User → Exploitation | 1 : 0..* |
| User → Notification | 1 : 0..* |
| User → Conseil | 1 : 0..* |
| User → Admin | 0..1 : 1 |
| Categorie → Produit | 1 : 0..* |
| Commande → LigneCommande | 1 : 0..* |
| Commande → Paiement | 1 : 0..1 |
| LigneCommande → Produit | N : 1 |
| Exploitation → SuiviCulture | 1 : 0..* |
| Exploitation → Rendement | 1 : 0..* |
