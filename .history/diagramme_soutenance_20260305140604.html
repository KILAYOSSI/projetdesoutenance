Relations entre les Classes

Utilisateur → Kyc : 1 utilisateur soumet 0 à plusieurs KYC
Utilisateur → Produit : 1 producteur publie 0 à plusieurs produits
Utilisateur → Commande : 1 client passe 0 à plusieurs commandes
Utilisateur → Exploitation : 1 agriculteur possède 0 à plusieurs exploitations
Utilisateur → Notification : 1 utilisateur reçoit 0 à plusieurs notifications
Utilisateur → Conseil : 1 utilisateur rédige 0 à plusieurs conseils
Utilisateur → Administrateur : 0 ou 1 utilisateur est un administrateur
Categorie → Produit : 1 catégorie contient 0 à plusieurs produits
Commande → LigneCommande : 1 commande contient 0 à plusieurs lignes
Commande → Paiement : 1 commande possède 0 ou 1 paiement
LigneCommande → Produit : plusieurs lignes concernent 1 produit
Exploitation → SuiviCulture : 1 exploitation a 0 à plusieurs suivis
Exploitation → Rendement : 1 exploitation a 0 à plusieurs rendements

Légende des Statuts

Kyc.statut : en_attente, valide, rejete
Commande.statut : en_attente, payee, en_cours, livree, annulee
Paiement.statut : en_attente, valide, echoue
Exploitation.statut : active, inactive
